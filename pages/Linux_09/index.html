09.静态、动态链接

# 静态连接和动态链接的实践性理解

# 一、抽象地理解静态链接和动态链接

静态链接：编译时把“别人写好的代码”直接拷贝进你的程序里 动态链接：编译时只记下“我以后要用谁”，真正的代码等程序运行时再找

注意这两个词的主语：

  • 静 / 动，说的不是程序本身
  • 说的是：库代码“什么时候、以什么方式”进入你的程序

# 🔹 用“做菜”打个不低级的比方

# 静态链接（static）

你做饭时：把 盐、油、酱油、锅全部装进你自己的背包，去到哪儿，都用你自己的

👉 好处：不依赖环境并且放到哪都能跑（嵌入式、固件喜欢）

👉 坏处：背包很重（文件大），每个人都背一套（浪费内存）

# 动态链接（dynamic）

你做饭时：背包里不带锅，只写一张纸：“我要用 3 号厨房的锅”

等你真正开始做饭（程序运行）：系统把锅给你，多个人还能共用同一口锅

👉 好处：程序小，内存可共享

👉 坏处：厨房没锅，你就完蛋（运行时报错）

# 三、静态链接：代码“拷贝”进可执行文件

静态链接的核心只有一句话：

目标文件里的代码，被直接合并进最终 ELF。

以下是打包并编译输出的过程：

ikun@ubuntu2004:~/Desktop/link_test$ gcc -c math_lib.c -o math_lib.o	# 生成目标文件
ikun@ubuntu2004:~/Desktop/link_test$ file math_lib.o	# 查看目标文件类型
math_lib.o: ELF 64-bit LSB relocatable, x86-64, version 1 (SYSV), not stripped # 这是一个还没被链接的“零件”
ikun@ubuntu2004:~/Desktop/link_test$ ar rcs libmath.a math_lib.o	# 把.o文件打包成.a文件，libmath这个名字是找math相关的文件

编写test_lib.c文件用于测试

#include <stdio.h>
#include "math_lib.h"

int main() {
    int a = 5, b = 3;
    printf("a + b = %d\n", add(a, b));
    printf("a * b = %d\n", mul(a, b));
    return 0;
}

ikun@ubuntu2004:~/Desktop/link_test$ gcc test_lib.c -o test_static -static -L. -lmath

  • -static选项表示生成静态链接，作用是把程序运行需要的所有依赖打包，如果不加，则为动态库
  • -L. 表示优先在当前目录下查找
  • -lmath表示生成的文件名，规则是自动在前面加lib，在后面加.a

# 四、动态链接：符号“引用”与运行时解析

动态链接并不拷贝代码，而是留下“欠条”。

ELF 里只记录：“我需要一个叫 add 的符号，它在某个共享库里，至于在哪里，俺不中嘞”

首先仍旧成成目标文件

ikun@ubuntu2004:~/Desktop/link_test$ gcc -c math_lib.c -o math_lib_pico.o -fPIC
1

-fPIC 的意义:-fPIC 让代码不依赖固定地址，通过 GOT/PLT 间接寻址，使同一份代码能被多个进程共享

ikun@ubuntu2004:~/Desktop/link_test$ gcc -shared -o libmath.so math_lib_pico.o
ikun@ubuntu2004:~/Desktop/link_test$ file libmath.so
libmath.so: ELF 64-bit LSB shared object, x86-64, version 1 (SYSV), dynamically linked, BuildID[sha1]=f701bf17254bed35c1ab68eea4487c995377e054, not stripped
1
2
3

-shared选项 ：不要生成可执行程序，生成“可被加载的共享对象”。

它会让链接器：不要求 main，允许未解析符号，生成 .so 类型 ELF

生成动态库后查一下户口（依赖关系、把文件扒开看看里面有哪些函数）

ikun@ubuntu2004:~/Desktop/link_test$ ldd test_dynamic
	linux-vdso.so.1 (0x00007ffd9bdf4000)
	libmath.so => ./libmath.so (0x0000748d6e61a000)	# 这一个库为用户打包的库
	libc.so.6 => /lib/x86_64-linux-gnu/libc.so.6 (0x0000748d6e200000)
	/lib64/ld-linux-x86-64.so.2 (0x0000748d6e626000)

ikun@ubuntu2004:~/Desktop/link_test$ export LD_LIBRARY_PATH=./:$LD_LIBRARY_PATH
