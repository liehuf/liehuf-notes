08.编译原理

# C语言编译过程

# 一、 编译 C 程序流程概况

  • GCC 编译的4 个阶段

    1. 预处理

      -E
      1
      • 宏展开、头文件展开、删除注释、条件编译处理
      • 输出：.i 文件

    2. 编译

      -S
      1
      • 语法语义分析 → 生成汇编代码
      • 输出：.s 文件

    3. 汇编

      -c
      1
      • 汇编 → 机器码，生成目标文件
      • 输出：.o 文件（ELF 格式，不可直接执行）

    4. 链接（无参数）

      -o
      1
      • 符号解析、重定位 → 生成可执行文件

  • 常用编译参数

    • -o：指定输出文件名
    • -Wall：打开所有警告
    • -g：添加调试信息（GDB 用）
    • -O1/-O2/-O3：优化等级（嵌入式非常关键）
    • -I：指定头文件路径
    • -L：指定库文件路径
    • -l：链接时指定库名

# 二、实操过程和细节（默认已有gcc工具链）

操作的.c对象

#include <stdio.h>

#define MAX_NUM 100          // ① 预处理会展开

int global_init = 10;        // ② .data
int global_uninit;           // ③ .bss

void print_info() {
    int local_var = MAX_NUM; // ④ 栈
    printf("global_init = %d, local_var = %d\n", global_init, local_var);
}

int main() {
    print_info();
    return 0;
}

# 阶段 1：预处理（-E）——纯文本世界

gcc -E test.c -o test.i
1

这一步不会生成任何二进制。

通过cat和grep命令可以看到一些信息

总结一下：

#define MAX_NUM 100，已经彻底消失了，编译器后面的阶段根本不知道“宏”这个概念

#include <stdio.h>展开成几百行代码，所以会看到很多 __printf_chk、`builtin...

test.i 本质：还是 C 代码，但已经是“展开后的、最原始”的 C

关键理解

  • test.s 是汇编语言文本
  • 编译器已经：
    • 决定了调用约定

# 阶段 3：汇编（-c）——真正的机器码，但还不能跑

gcc -c test.s -o test.o
1

验证一下：

ikun@ubuntu2004:~/Desktop/c_test$ file test.o
test.o: ELF 64-bit LSB relocatable, x86-64, version 1 (SYSV), not stripped
1
2

通过 file 命令看到 test.o 是 "ELF 64-bit LSB relocatable, not stripped"，这说明它是在 Linux 下的标准 64 位半成品文件，需要链接才能运行；而 "not stripped" 意味着它保留了函数名和行号等调试信息，让你能用 GDB 调试工具清楚地看到代码执行到哪里，方便排查错误。

重点是段结构：

ikun@ubuntu2004:~/Desktop/c_test$ readelf -S test.o | grep -E "\.text|\.data|\.bss|\.symtab"

  [11] .symtab           SYMTAB           0000000000000000  00000160

段名 来自哪里
.text print_info、main 的机器码
.data global_init = 10
.bss global_uninit
.symtab 所有函数 / 变量名字

.bss：不占磁盘空间,只记录“要分配多少内存”,程序加载时由内核清零

# 阶段 4：链接 —— 一切才“真的能跑”

gcc test.o -o test
1

gcc 实际做了：ld + crt + libc + 重定位

这时候就可以运行了

ikun@ubuntu2004:~/Desktop/c_test$ ./test
global_init = 10, local_var = 100 # 哦，我的老天鹅，终于把你盼出来了

2

连接器干了两件事

  1. 符号解析
    • printf → libc 里的地址
    • __libc_start_main → 程序入口
  2. 重定位
    • 把 .o 里 %rip + offset
    • 改成加载后能用的地址

💡 没有链接器，CPU 根本不知道“你在 call 谁”。
