## 一、Makefile的基本规则

Makefile规则长如下的样子：

```makefile
目标(target): 依赖(prerequisites)
	命令(command)
```

写Makefile时，必须注意一下三个规则

- 命令前面缩进必须是`Tab`
- 命令是真的要执行的Shell命令
- 是否执行命令得看时间戳

通过如下的例子完整走一遍脑子：

```makefile
app: test.o math_lib.o
	gcc -o app test.o math_lib.o
```

Make 做的事情是：

1. 我需要 `app`
2. `app` 依赖 `test.o` 和 `math_lib.o`
3. 那我先去看看：``test.o` 怎么来？`math_lib.o` 怎么来？

于是这货向下找，啊，找到你了！

```makefile
test.o: test.c
	gcc -c test.c -o test.o
```

## 二、规则的三大类

### 普通规则

```makefile
test.o: test.c
	gcc -c test.c -o test.o
```

一个目标一个规则，简单清晰，文件一多就爆炸，靠近一点快被融化

高端的仁一般不这样写，但是我是二般的来着

### 模式规则（这才是主要的生产力）

譬如：

```makefile
%.o: %.c
	$(CC) $(CFLAGS) -c $< -o $@
```

`%` = **文件名中“变化的那一部分”**test.c → test.o

“任何 `.o` 文件，都可以由**同名的 `.c` 文件**通过这条命令生成”，这给了Make一条生成方法

其中：

- \$<   # 第一个依赖（xxx.c）
- $@   # 当前目标（xxx.o）

###  伪目标（.PHONY，工程经验点）

当我有一个文件`clean`，然后我在Makefile里面写道：

```makefile
clean:
	rm -f *.o
```

然后make clean一下，结果，Make以为都有了，那我啥都不干了！😵‍💫

其实正确的姿势为：

```makefile
.PHONY: clean

clean:
	$(RM) *.o
```

`.PHONY` 的意思是：

> **“这个目标不是文件名，它只是一个命令入口。”**

## 三、其他的注意点

1. 最上面的总是默认依赖，有一个约定俗成的 `all: app`
2. 一个规则可有多个依赖，但反过来不行
3. Makefile的执行顺序并非自上而下，而是依赖规则
4. 在命令前面加一个`@`就可以让命令行停止叽叽喳喳了



一般的优雅Makefile长这样：

一句话定义：

> **优雅 = 不重复、可扩展、改动成本低**

```makefile
CC := gcc
CFLAGS := -Wall -g

OBJ := main.o add.o

app: $(OBJ)
	$(CC) -o $@ $^

%.o: %.c
	$(CC) $(CFLAGS) -c $< -o $@
```

这里有必要解释一些点：

1. `CFLAGS := -Wall -g`两个选项之中，
   1. 第一个`-Wall`是Warn all，代表几乎所有常见警告
   2. 第二个`-g`代表生成调试信息
2. 命令`gcc -o app main.o add.o`的作用不是编译，而是打包，把 **已经编译好的目标文件** `main.o` 和 `add.o`**链接** 成一个可执行文件 `app`
3. `%.o: %.c`代表模式匹配，这一点有点像广义的`for`循环，这里的`%`代表同名占位符
