## 一、交叉编译的结构基础

一、什么是 Host？什么是 Target？

- **Host**：正在进行编译操作的机器
- **Target**：最终运行程序的机器

在当前场景中：

- Ubuntu22.04（x86_64）= Host
- 香橙派（ARM）= Target

关键问题：

为什么在 Ubuntu 上 `gcc test.c` 编出来的程序，不能直接在 ARM 上运行？

答案：**CPU 指令集不同。**

------

## 二、指令集架构（ISA）

CPU 不只是“算东西的盒子”，它理解的是特定的指令编码。

- x86_64 → Intel / AMD 架构
- ARM → 嵌入式常见架构

同一段 C 代码：

```
int main() {
    return 0;
}
```

在不同架构上，会被翻译成完全不同的机器码。

机器码不同 → CPU 无法识别 → 无法执行。

------

## 三、什么是 ELF？

Linux 下的可执行文件，本质是 **ELF 文件**（Executable and Linkable Format）。

ELF 头部会明确记录：

- 架构类型
- 字节序
- ABI
- 入口地址

这就是我们验证 Host / Target 的关键工具。
